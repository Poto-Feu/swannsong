Save file format

The save file is a binary file divided into 3 parts:

  • Header
  • Chunks
  • Checksum

Header

The header contains several elements in the following order:

  1. A 4-char program signature (SWSA)
  2. A 4-char file type signature (SAVE)
  3. A 16-bit unsigned int storing file type major version
  4. A 16-bit unsigned int storing file type minor version

Chunks

A save file chunk is composed of several elements in the following order:

  1. A 16-bit unsigned int enum value corresponding to the chunk type - enum values are defined in save_const.hpp
  2. A 32-bit unsigned int value corresponding to the size of the chunk content
  3. Content bits

There is no separator between chunks since their size is specified.

Checksum

A SHA-1 checksum is calculated from the file data (including the header) to check its integrity. It is appended at the end of the save file.
